자판기 시뮬레이션입니다.
구매자 및 판매자로써 시뮬레이션을 할 수 있습니다.
구매자는 돈을 넣고 거스름돈을 받는 과정과, 판매자는 재고 관리 및 물품 설정 등을 할 수 있습니다.
UI 및 일러스트 디자인과 부드러운 애니메이션 효과를 더했습니다.


 대학교 재학 중, Java로 만들어진 자판기 프로그램을 제작하는 것이 목표인 과제를 받았습니다.
단기간이지만 기본을 탄탄하게 설계하고, 시각적으로도 짜임새 있게 만들고자 하였습니다.

UI DESIGN

UI 디자인은 심플하게 구성하였습니다. 일러스트를 직접 만들어 사용하였고, ease 함수로 여러 부분에 애니메이션을 적용하였습니다.

HOW MADE ?

시스템의 구조

시스템의 흐름을 총괄하는 Game 클래스가 매 프레임마다 각 Page 클래스를 업데이트하고, 각 Page는 필요한 오브젝트를 생성하고, 이 오브젝트들을 업데이트 시킵니다.

Game 클래스의 Run이 호출될 때, 초당 60프레임으로 반복되는 루프를 작동시키도록 만들었습니다. 루프가 종료된 직후 render 함수를 호출하여 화면에 그래픽 요소들을 그립니다.

루프가 반복될 때마다 현재 페이지의 Update 함수를 호출함으로써, 해당 페이지도 Update 함수를 통해 매 프레임마다 반복되는 루프의 기능을 사용할 수 있습니다.

render 함수 부분에서는 createBufferStrategy 함수를 사용하여 트리플 버퍼링을 구현하여 화면의 깜빡임(flickering)을 방지하였습니다.

또한, 전면에 백그라운드를 그려주고 안티엘리어싱을 적용한 후, 현재 페이지의 Render 함수를 호출하여 해당 함수를 통해 그래픽 요소들을 그릴 수 있게 하였습니다.

구매자는 돈을 넣어야만 자판기의 음료들을 구매할 수 있습니다. 또한, 거스름돈도 반환받아야 합니다. 이에 돈을 각 오브젝트 클래스로 만들어 조작할 수 있게 하였습니다.

coinNumber변수는 모든 화폐의 총 갯수입니다. 위의 getCoinNum 함수는 이를 이용하여 모든 화폐중 인자값과 같은 값인 화폐의 갯수를 반환합니다.

removeCoinNum 함수는 입력한 값을 가진 코인을 num개 만큼 제거합니다.
자판기가 돌려줄 거스름돈들을 반환할 때 사용합니다.

 Game 클래스의 구조입니다. Intialize 함수는 main 함수에서 맨 처음 딱 한번 호출됩니다. 이 함수에서는 SDL 윈도우 및 랜더러를 생성하고, BGM을 불러오고, 유저 데이터가 존재하는지 등을 판단합니다.

 RunLoop 함수는 게임이 진행되는 동안 키가 눌리는 것을 인식하는 ProcessInput, 게임 내 오브젝트들을 프레임 단위로 업데이트하는 UpdateGame, 화면에 보여주는 것을 담당하는 GenerateOutput 함수 순서로 무한루프됩니다.

 또한 기본적으로 게임의 프레임 단위의 실행을 제어합니다. 델타 타임을 16ms로 조절하여 프레임을 제한합니다. 이로 하여 1초에 약 60프레임 정도를 실행할 수 있습니다.

 UpdateGame 함수에서 현재 룸의 Update 함수를 호출하여 룸이 Update가 가능하도록 해줍니다.
또, 룸을 이동할 때 나타나는 스크린 애니메이션 효과를 보여주는 오브젝트를 관리하기도 합니다.

 GenerateOutput 함수에서는 현재 룸의 Draw 함수를 호출하여 룸의 요소들을 그려줍니다.



룸 시스템

 오브젝트들이 배치되는 구역을 나누는 시스템으로, 이 구역을 룸이라고 표현하였습니다. 각 룸 오브젝트는 해당 룸에 속해있는 오브젝트들을 할당 및 관리하고 룸에 UI를 표시하고 작동되도록 합니다.

 룸들의 파일 시스템을 위와 같이 정렬하여 간편하게 관리합니다. 게임을 처음 실행하여 유저 데이터가 없는 클라이언트는 Tutorial 룸이 처음 할당됩니다.

 모든 룸들은 기본적으로 인풋 함수들과 Update, Draw, Delete 함수들을 가지고 있습니다. 또한, 각 룸마다 다른 기능들을 가진 변수들을 선언하고 초기화합니다.
