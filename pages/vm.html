VENDING MACHINE


플랫폼 - PC
개발환경 - JAVA
개발일 - 2020.11

자판기 시뮬레이션입니다.
구매자 및 판매자로써 시뮬레이션을 할 수 있습니다.
구매자는 돈을 넣고 거스름돈을 받는 과정과, 판매자는 재고 관리 및 물품 설정 등을 할 수 있습니다.
UI 및 일러스트 디자인과 부드러운 애니메이션 효과를 더했습니다.


개발 배경

대학교 재학 중, Java로 만들어진 자판기 프로그램을 제작하는 것이 목표인 과제를 받았습니다.

단기간이지만 기본을 탄탄하게 설계하고, 시각적으로도 짜임새 있게 만들고자 하였습니다.

개발 이후, 포트폴리오를 포스팅 하며 부족한 부분들을 찾게되었고, 깃허브에 올려 오픈소스화 하였습니다.

Github - https://github.com/PlaneGames/VendingMachine

주요 요구 사항

판매하는 음료는 각각 물(450원), 커피(500원), 이온음료(550원), 고급커피(700원), 탄산음료(750원)이다.

각각 음료의 재고는 기본적으로 3개, 품절시 품절 표시.

입력 가능 화폐 단위는 10원, 50원, 100원, 500원, 100원이다.

한번에 지폐로는 3000원 까지, 동전을 모두 포함하여 5000원까지만 입력이 가능하다.

UI DESIGN

UI 디자인은 심플하게 구성하였습니다. 일러스트를 직접 만들어 사용하였고, ease 함수로 여러 부분에 애니메이션을 적용하였습니다.

HOW MADE ?

시스템의 구조

시스템의 구조를 설계함에 앞서, 이전부터 사용했던 Gamemaker라는 엔진에서 영감을 얻을 수 있었습니다. 게임메이커는 오브젝트들을 룸에 배치하여 해당 룸에 속한 오브젝트들을 보여주는 구조로 되어있습니다. 이 구조의 장점은 구축하기 쉽고, 관리하기도 용이하다는 점이었습니다. 이 구조를 구현해 보기로 하였습니다.

시스템의 흐름을 총괄하는 Game 클래스가 매 프레임마다 각 Page 클래스를 업데이트하고, 각 Page는 필요한 오브젝트를 생성하고, 이 오브젝트들을 업데이트합니다.

Game 클래스의 Run이 호출될 때, 초당 60프레임으로 반복되는 루프를 작동시키도록 만들었습니다. 루프가 종료된 직후 render 함수를 호출하여 화면에 그래픽 요소들을 그립니다.

루프가 반복될 때마다 현재 페이지의 Update 함수를 호출함으로써, 해당 페이지도 Update 함수를 통해 매 프레임마다 반복되는 루프의 기능을 사용할 수 있습니다.

Game 클래스의 render 함수 부분에서는 createBufferStrategy 함수를 사용하여 트리플 버퍼링을 구현하여 화면의 깜빡임(flickering)을 방지하였습니다.

또한, 전면에 백그라운드를 그려주고 텍스트에 안티엘리어싱을 적용해 픽셀을 부드럽게 그려줄 수 있게 하고, 현재 페이지의 Render 함수를 호출하여 해당 함수를 통해 그래픽 요소들을 그릴 수 있게 하였습니다.



자판기 돈 입력 시스템

자판기에 돈을 넣기위해 만든 버튼들을 각각의 클래스로 구현하였고, 기능들 또한 각각 정의하였었습니다. 그러나, 리소스의 낭비와 기능 수정시 일일이 모든 클래스를 수정해야 할 것 같았습니다.

이에, ObjCoinButtonPar 라는 부모 클래스를 만들어 공통된 기능들을 전부 묶고, 각 버튼 클래스들은 위치 Offset과 이미지 설정만 한 뒤, 기능들을 상속받아 사용하였습니다. 덕분에 버튼들에 마우스 호버 효과 등, 애니메이션 효과들을 보다 쉽게 구현할 수 있었습니다.

한번 구매시 자판기가 지폐로 입력받는 돈의 상한선은 3,000원, 지폐와 동전을 포함하여 총 5,000원을 초과할 수 없도록 조건을 걸었습니다.

자판기에 넣을 수 있는 돈에는 1000원짜리 지폐와, 동전들이 있습니다. 돈 버튼을 누를경우 해당 돈의 coin 오브젝트를 생성하고 리스트에 입력합니다.

이 동전 입력 버튼들은 ObjCoinButtonPar 클래스의 자식 클래스입니다. ObjCoinButtonPar에서 위치별 마우스 클릭 인식 범위 지정과, 클릭시 자판기에 돈이 입력되는 기능 등을 정의합니다.



자판기 거스름돈 시스템

자판기는 기본적으로 잔돈을 각 동전별로 5개씩 가지고 있도록 하였습니다.

구매자는 돈을 넣어야만 자판기의 음료들을 구매할 수 있습니다. 또한, 거스름돈도 반환받아야 합니다. 이에 돈을 각 오브젝트 클래스로 만들어 조작할 수 있게 하였습니다.

coinNumber변수는 모든 화폐의 총 갯수입니다. 위의 getCoinNum 함수는 이를 이용하여 모든 화폐중 인자값과 같은 값인 화폐의 갯수를 반환합니다.

removeCoinNum 함수는 입력한 값을 가진 코인을 num개 만큼 제거합니다.
자판기가 돌려줄 거스름돈들을 반환할 때 사용합니다.

거스름돈에 필요한 동전의 종류와 수를 계산하는 부분에 있어서, 어떤식으로 구현해야할지 고민되었습니다. 처음에는 조건문으로 계산하려고 하였으나 뜻대로 되지 않았습니다. 이에 650원의 거스름돈으로 가정해보고 500으로 나누기 및 mod 연산을 해보았습니다.

650 / 500 = 1.3에서 정수부는 필요한 500원의 갯수, 650 % 500 = 150의 값은 500원으로 해결되지 않는 나머지 값이었습니다. 나머지 값인 150을 100으로 나누고, 이 과정을 10원까지 반복해 원하는 결과를 얻게 되었습니다.



마치며

과제에서 필요한 자판기의 요구 조건들을 만족하였지만 여기에는 거슬러줄 동전이 부족할 때 발생하는 논리적 오류에 대한 예외 상황이 없었습니다.

포트폴리오에 포스팅하기 위해 소스코드를 다시 살펴보았는데, 이러한 부분들에 리팩토링 및 기능 추가가 필요해 보였습니다.

이에 깃허브에 이 프로젝트를 오픈소스로 공유하기로 하였고, 아직 다 채워지지 않은 기능들을 추가해볼 생각입니다.
