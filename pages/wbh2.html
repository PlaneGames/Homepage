WAY BACK HOME 2

플랫폼 - PC
개발환경 - C++ SDL

약간의 스토리가 가미된 블록 퍼즐 게임입니다. 숫자가 정확히 0이 될 때 목적지에 도달해야합니다. 전작과는 다르게 지나온 길을 다시 이동할 수 있지만, 한번 밟으면 없어지는 바닥을 추가하였습니다.

GAME DESIGN

 게임의 디자인은 깔끔한 픽셀 그래픽으로 디자인 되었습니다. 부드러운 애니메이션 효과들을 여럿 사용하여 게임이 전체적으로 몽글몽글한 느낌을 주도록 하였습니다.

MAP DESIGN

 스테이지를 거듭할 수록 맵의 형태가 다양해지고 복잡해지도록 설계되어있습니다.

MAP EDITOR

 게임 내부에 맵 에디터를 만들었습니다. 언제든지 만들고 테스트가 가능합니다. .pmap 확장자 파일로 저장 및 불러오기가 가능합니다.

HOW MADE ?

게임 시스템의 구조

 Game 클래스에서 시스템의 전체적인 흐름을 총괄합니다.

 main 함수에서 Game 클래스를 메모리에 할당하고, 정상적으로 처리되면 Game 클래스를 통해 무한루프가 실행됩니다. 무한루프가 끝날경우 메모리를 제거합니다.

 Game 클래스의 구조입니다. Intialize 함수는 main 함수에서 맨 처음 딱 한번 호출됩니다. 이 함수에서는 SDL 윈도우 및 랜더러를 생성하고, BGM을 불러오고, 유저 데이터가 존재하는지 등을 판단합니다.

 RunLoop 함수는 게임이 진행되는 동안 키가 눌리는 것을 인식하는 ProcessInput, 게임 내 오브젝트들을 프레임 단위로 업데이트하는 UpdateGame, 화면에 보여주는 것을 담당하는 GenerateOutput 함수 순서로 무한루프됩니다.

 또한 기본적으로 게임의 프레임 단위의 실행을 제어합니다.
델타 타임을 16ms로 조절하여 프레임을 제한합니다. 이로 하여 1초에 약 60프레임 정도를 실행할 수 있습니다.

 UpdateGame 함수에서 현재 룸의 Update 함수를 호출하여 룸이 Update가 가능하도록 해줍니다.
또, 룸을 이동할 때 나타나는 스크린 애니메이션 효과를 보여주는 오브젝트를 관리하기도 합니다.

 GenerateOutput 함수에서는 현재 룸의 Draw 함수를 호출하여 룸의 요소들을 그려줍니다.



룸 시스템

 오브젝트들이 배치되는 구역을 나누는 시스템으로, 이 구역을 룸이라고 표현하였습니다. 각 룸 오브젝트는 해당 룸에 속해있는 오브젝트들을 할당 및 관리합니다.

 룸들의 파일 시스템을 위와 같이 정렬하여 간편하게 관리합니다. 게임을 처음 실행하여 유저 데이터가 없는 클라이언트는 Tutorial 룸이 처음 할당됩니다.

 모든 룸들은 기본적으로 인풋 함수들과 Update, Draw, Delete 함수들을 가지고 있습니다. 또한, 각 룸마다 다른 기능들을 가진 변수들을 선언하고 초기화합니다.
