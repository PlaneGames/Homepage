TYPING OF LEGEND


플랫폼 - PC
개발환경 - C Win32 api
개발일 - 2019.9 ~ 2019.10

타자 연습 게임입니다.
간단한 문장연습과, 순간 기억이라는 컨텐츠를 가지고 있습니다.
Winapi를 사용하여 깔끔한 그래픽을 구축하였고, 이를 이용하여 유저의 정보와 타자 변화를 보여주는 그래프를 제공합니다.

개발 배경

대학교 과제로, C언어로 타자연습 프로그램을 제작하는 미션을 받게 되었습니다.

기존에 C언어를 배울 때 콘솔창만을 사용하는 경우가 많았는데, 프로젝트 만큼은 C언어로 깔끔한 그래픽을 구축하고 싶었습니다.

이에, win32 api를 배우면서 사용해보고자 하였습니다.

UI DESIGN

UI 디자인은 win32 api와 GDI+ 라이브러리를 사용하여 그래픽 요소들을 심플하게 구성하였습니다. 대표적으로 유저 정보 창에 그래프를 색깔별로 그려 넣어보았습니다.
테스트를 위하여 날짜별 타자 정보를 임의로 채워 넣었습니다.

HOW MADE ?

타자 입력 시스템

문장 타자 연습을 만들 때, 목표 텍스트와 입력중인 텍스트를 비교하여 정확도를 계산하고자 하였습니다.
처음 생각과는 달리, 목표 텍스트와 입력된 텍스트가 같은지의 여부만을 알 수 있었습니다. 또한, 틀린 글자만 빨간색으로 처리하는 부분이 불가능 하였습니다.


그래서, 반복문을 이용한 문자 단위의 비교 설계의 필요성을 느끼게 되었습니다.

traslateText변수에는 목표 텍스트의 i번째 문자, traslateText2변수에는 입력중인 텍스트의 i번째 문자를 가집니다.
반복문을 이용하여 각 문자별로 비교를 진행합니다.

각 문자별로 출력해야 하므로, 각 문자가 정확한 위치에 그려져야 했습니다.
이에 현재 i번째 문자의 종류별로 다른 크기값을 구하고 xx 변수에 더하도록 하였습니다.

최종적으로 DrawString 함수를 통해 입력중인 traslateText 문자가 해당 좌표로 그려지게 됩니다.



사용자 정보 인터페이스

사용자 정보 인터페이스를 구현하기 위해 GDI+ 라이브러리에 속한 함수들을 다수 사용하고자 하였습니다.


하지만, 단순히 함수들을 사용하여 구현하기에는 리소스의 양이 너무 많아질 것 같았습니다. 이에 헤더파일에 그래픽 함수들을 미리 만들게 되었습니다.

대표적인 함수로 draw_text_color_rect가 있습니다.


DrawString 함수에, 정렬, 안티엘리어싱, 색조 변경의 기능을 더해 한번의 함수 호출로 사용하고 싶어서 설계하였습니다.


그래픽 라이브러리를 보다 쉽게 활용하기 위해, 위와 같은 함수를 10가지 넘게 설계하였습니다.
비록 오래된 라이브러리에 트랜드있는 기술은 아니지만,
충분히 활용 가능하고, 작품을 만들어볼 수 있어서 좋은 경험이었습니다.
